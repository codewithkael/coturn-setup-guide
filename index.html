Coturn Setup Guide

A polished, production-friendly setup guide for configuring Coturn on a VPS.

Overview

This document walks through:

  • Creating a non-root sudo user.
  • Installing Coturn and Certbot.
  • Setting up DNS for turn.codewithkael.com.
  • Issuing a Let’s Encrypt certificate.
  • Configuring /etc/turnserver.conf.
  • Giving Coturn permission to bind port 443.
  • Starting and validating the service.
  • Using the TURN server in a WebRTC app.

Step 0 — SSH in as root

ssh root@YOUR_SERVER_IP

Use your provider’s normal password or SSH key setup first.

Step 1 — Create a non-root user with sudo

adduser kael
usermod -aG sudo kael
echo "kael ALL=(ALL) NOPASSWD:ALL" >/etc/sudoers.d/kael
chmod 440 /etc/sudoers.d/kael
exit
ssh kael@YOUR_SERVER_IP

Replace kael with your preferred username.

Step 2 — Install basic packages and Coturn

sudo apt update
sudo apt install -y coturn certbot
sudo systemctl stop coturn

Step 3 — DNS and Cloudflare setup for turn.codewithkael.com

In Cloudflare, create an A record:

  • Name: turn
  • Type: A
  • Value: YOUR_SERVER_IP
  • Proxy status: DNS only (gray cloud)

Then verify propagation:

dig +short turn.codewithkael.com

Step 4 — Issue a Let’s Encrypt certificate

sudo lsof -i :80 || echo "no process on 80"
sudo certbot certonly --standalone -d turn.codewithkael.com
ls -l /etc/letsencrypt/live/turn.codewithkael.com

You should see fullchain.pem and privkey.pem symlinks.

Step 5 — Adjust certificate permissions

sudo chmod 755 /etc/letsencrypt
sudo chmod 755 /etc/letsencrypt/live
sudo chmod 755 /etc/letsencrypt/archive
sudo chmod 644 /etc/letsencrypt/archive/turn.codewithkael.com/privkey1.pem

If the filename differs, update the privkey*.pem path accordingly.

Step 6 — Create /etc/turnserver.conf

sudo nano /etc/turnserver.conf

Use this minimal working configuration (3478 plain, 443 TLS):

# Ports
listening-port=3478
alt-listening-port=0

tls-listening-port=443
alt-tls-listening-port=0

# IPs
listening-ip=YOUR_SERVER_IP
external-ip=YOUR_SERVER_IP

# Relay ports
min-port=49152
max-port=65535

# Auth: static user/pass
lt-cred-mech
realm=turn.codewithkael.com
user=username1:password1

# TLS certs
cert=/etc/letsencrypt/live/turn.codewithkael.com/fullchain.pem
pkey=/etc/letsencrypt/live/turn.codewithkael.com/privkey.pem

# Security
no-rfc5780
no-stun-backward-compatibility
response-origin-only-with-rfc5780

# Logging
syslog

Step 7 — Ensure Coturn runs with this config

Check the service unit:

grep ExecStart /usr/lib/systemd/system/coturn.service 2>/dev/null || \
  grep ExecStart /lib/systemd/system/coturn.service 2>/dev/null

If it shows ExecStart=/usr/bin/turnserver -c /etc/turnserver.conf --pidfile=, that’s fine.

Step 8 — Allow Coturn to bind port 443

Give the turnserver binary permission to bind privileged ports like 443 while still running as the unprivileged user:

sudo setcap 'cap_net_bind_service=+ep' /usr/bin/turnserver
getcap /usr/bin/turnserver

Expected output:

/usr/bin/turnserver = cap_net_bind_service+ep

Step 9 — Enable and start Coturn

sudo sed -i 's/#TURNSERVER_ENABLED=1/TURNSERVER_ENABLED=1/' /etc/default/coturn
sudo systemctl restart coturn
sudo systemctl status coturn
sudo ss -tulpen | grep -E '3478|443'

You want:

  • UDP + TCP on 3478
  • UDP + TCP on 443 (for TURN over TLS/DTLS)

Step 10 — Use in your WebRTC app

iceServers: [
  { urls: "stun:turn.codewithkael.com:3478" },
  {
    urls: [
      "turn:turn.codewithkael.com:3478?transport=udp",
      "turn:turn.codewithkael.com:3478?transport=tcp",
      "turns:turn.codewithkael.com:443?transport=tcp"
    ],
    username: "username1",
    credential: "password1"
  }
]

Notes

  • Replace YOUR_SERVER_IP everywhere it appears.
  • Keep the turn DNS record set to DNS only.
  • Test connectivity after restarting Coturn.
